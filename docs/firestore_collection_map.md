# BimariHaunter: Comprehensive Firestore Database Collection Map & Schema Specification

This document provides a complete, production-grade schema blueprint and hierarchical map of all **Google Cloud Firestore** collections, subcollections, fields, data types, and document structures for the **BimariHaunter** platform. 

Pass this technical specification directly to the Android client agent (**Antigravity**) to ensure the native Kotlin Room cache, Firebase SDK bindings, and data deserializers match the cloud database with 100% precision.

---

## 1. Global Firestore Collection Hierarchy

```
/ (Database Root)
├── users (Collection)
│   └── {userId} (Document - ID is Firebase Auth User UID)
│       ├── feed (Subcollection - FIFO Stack capped at 50)
│       │   └── {reportId} (Document - ID is deterministic SHA-256 hash)
│       ├── friends (Subcollection - Mutual Friend List)
│       │   └── {friendId} (Document - ID is friend's UID)
│       └── blocked (Subcollection - Blocked User List)
│           └── {blockedId} (Document - ID is blocked user's UID)
│
├── reports (Collection - Global Outbreak Database)
│   └── {reportId} (Document - ID is deterministic SHA-256 hash)
│
├── chats (Collection - AI Advisor Chat Threads)
│   └── {chatId} (Document - ID is auto-generated by Firestore SDK)
│       └── messages (Subcollection - Conversation History)
│           └── {messageId} (Document - ID is auto-generated by Firestore SDK)
│
├── groups (Collection - Regional Community Forums)
│   └── {groupId} (Document - ID is auto-generated by Firestore SDK)
│       └── messages (Subcollection - Community Bulletin Board Messages)
│           └── {messageId} (Document - ID is auto-generated by Firestore SDK)
│
└── direct_chats (Collection - Peer-to-Peer Chat Threads)
    └── {chatId} (Document - ID is deterministic sorted UID pair: `min(uid1,uid2) + "_" + max(uid1,uid2)`)
        └── messages (Subcollection - Message Stream with Actions)
            └── {messageId} (Document - ID is auto-generated or custom)
```

## 2. Comprehensive Collection Schemas

### 2.1 `/users` Collection
* **Document ID**: Strictly set to the user's authentic **Firebase Auth User UID** (`request.auth.uid`).
* **Purpose**: Manages the user's geolocated profile and personalized settings.

| Field Name | Firestore Data Type | Explanation / Constraint |
| :--- | :--- | :--- |
| `uid` | `String` | Matches the Firebase Auth User UID. |
| `email` | `String` | Registered user email address. |
| `name` | `String` | Registered user name (default: "Unknown"). |
| `live_location` | `Map` | Nested map containing geographical location details. |
| └── `city` | `String` | Capitalized city name (e.g. `"Karachi"`, `"Lahore"`) used to sync geolocated feeds. |
| └── `coordinates` | `GeoPoint` | Native Firestore `GeoPoint(latitude, longitude)` representing live user position. |
| `updated_at` | `Timestamp` | Automated server timestamp (`firestore.SERVER_TIMESTAMP`) of last profile update. |

#### **Example Document Blueprint (`/users/test_user_kotlin_999`)**:
```json
{
  "uid": "test_user_kotlin_999",
  "email": "test-kotlin@bimarihaunter.pk",
  "name": "Taha Bukhari Test",
  "live_location": {
    "city": "Karachi",
    "coordinates": {
      "_latitude": 24.8607,
      "_longitude": 67.0011
    }
  },
  "updated_at": "2026-05-20T06:40:00Z"
}
```

### 2.2 `/users/{userId}/feed` Subcollection
* **Document ID**: Deterministic **SHA-256 hash** of the original source article URL, matching the parent `/reports` document ID.
* **Purpose**: Personalized location-aware health feed. Maintained as a **FIFO stack limited to exactly 50 documents** by both user location changes and background scrapers.
* **Schema**: *Inherits exactly the schema of `/reports` (see Section 2.3 below).*

---

### 2.3 `/reports` Collection
* **Document ID**: Deterministic **SHA-256 hash** of the source article URL (prevents duplicate scraping).
* **Purpose**: Global database of crawled news and social media disease outbreak reports in Pakistan.

| Field Name | Firestore Data Type | Explanation / Constraint |
| :--- | :--- | :--- |
| `title` | `String` | Title of the outbreak article or head snippet of the social post. |
| `source` | `String` | Source publisher (e.g. `"Dawn News"`, `"Express Tribune"`, `"Twitter"`). |
| `url` | `String` | Original web source hyperlink. |
| `raw_text` | `String` | Extracted raw article text or post text. |
| `source_type` | `String` | Either `"web"` or `"social"`. |
| `status` | `String` | Outbreak processing status: `"analyzed"`, `"failed"`, or `"pending"`. |
| `published_at` | `Timestamp` | Original publication date. |
| `scraped_at` | `Timestamp` | Server date when the article was scraped and stored. |
| `ai_analysis` | `Map` | Rich NLP extracted entity markers. |
| ├── `disease` | `String` | Classified disease (e.g. `"dengue"`, `"malaria"`, `"cholera"`, `"polio"`). |
| ├── `severity` | `String` | Severity rating: `"high"`, `"medium"`, or `"low"`. |
| ├── `confidence_score`| `Number` | NLP classification probability float (e.g. `0.89`). |
| ├── `model_used` | `String` | AI model name (e.g. `"facebook/bart-large-mnli"`). |
| ├── `summary` | `Array [String]` | Key bullet points summarizing the outbreak details and instructions. |
| ├── `symptoms` | `Array [String]` | Extracted active symptoms (e.g. `["fever", "joint pain", "rash"]`). |
| ├── `locations` | `Array [String]` | Geocoded city and neighborhood locations (e.g. `["Karachi", "Clifton"]`). |
| └── `coordinates` | `GeoPoint` | Geocoded center coordinate marker `GeoPoint(latitude, longitude)`. |

#### **Example Document Blueprint (`/reports/sha256_dengue_hash_101`)**:
```json
{
  "title": "Dengue spike reported in Gulshan Karachi",
  "source": "Dawn News",
  "url": "https://dawn.com/article/dengue-karachi",
  "raw_text": "Gulshan-e-Iqbal town reports 15 new dengue admissions this week...",
  "source_type": "web",
  "status": "analyzed",
  "published_at": "2026-05-20T02:45:00Z",
  "scraped_at": "2026-05-20T03:00:00Z",
  "ai_analysis": {
    "disease": "dengue",
    "severity": "high",
    "confidence_score": 0.94,
    "model_used": "facebook/bart-large-mnli",
    "summary": [
      "Gulshan-e-Iqbal has reported 15 new dengue hospitalizations.",
      "Municipal authorities advise immediate mosquito net controls."
    ],
    "symptoms": ["fever", "joint pain", "headache"],
    "locations": ["Karachi", "Gulshan"],
    "coordinates": {
      "_latitude": 24.8922,
      "_longitude": 67.0747
    }
  }
}
```

### 2.4 `/chats` Collection
* **Document ID**: Auto-generated by the Firestore SDK (`DocumentReference.id`).
* **Purpose**: Houses AI chat sessions between the user and the advisor bot.

| Field Name | Firestore Data Type | Explanation / Constraint |
| :--- | :--- | :--- |
| `chat_id` | `String` | Copy of the document ID string. |
| `user_id` | `String` | Owner UID matching the authenticating user (`/users/{userId}`). |
| `title` | `String` | Chat thread topic (e.g. `"Karachi Dengue Inquiry"`, `"Offline Advisor Session"`). |
| `created_at` | `Timestamp` | Thread creation date (`firestore.SERVER_TIMESTAMP`). |
| `updated_at` | `Timestamp` | Date of the latest message (`firestore.SERVER_TIMESTAMP`) used for ordering lists. |

#### **Example Document Blueprint (`/chats/VLg1Zkvt9ZQ3Kk2Tm07C`)**:
```json
{
  "chat_id": "VLg1Zkvt9ZQ3Kk2Tm07C",
  "user_id": "test_user_kotlin_999",
  "title": "Karachi Dengue Inquiry",
  "created_at": "2026-05-20T06:42:00Z",
  "updated_at": "2026-05-20T06:42:30Z"
}
```

### 2.10 `/direct_chats` Collection
* **Document ID**: Deterministic lexicographically sorted combination: `min(uid1, uid2) + "_" + max(uid1, uid2)`.
* **Purpose**: Hosts peer-to-peer user direct messaging channels.

| Field Name | Firestore Data Type | Explanation / Constraint |

| `created_at` | `Timestamp` | Date of direct chat channel creation. |
| `participants` | `Array [String]` | User UIDs participating in the chat (exactly two). |
| `names` | `Map` | Lookup map mapping user UIDs to display names for instant top bar UI loads. |

## 3. Recommended Firestore Security Rules Configuration

To secure this exact hierarchy on Firebase so users can only access their own private information while keeping global maps/reports public, use this rules set:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // 1. Users Collection Rules
    match /users/{userId} {
      allow read: if request.auth != null; // Allows prefix searches for addition
      allow write: if request.auth != null && request.auth.uid == userId;
      
      // User's Subcollection geolocated Feed
      match /feed/{feedId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }

      // Mutual Friend list
      match /friends/{friendId} {
        allow read: if request.auth != null && request.auth.uid == userId;
        // Authenticated users can write mutual connections if friend exists
        allow write: if request.auth != null; 
      }

      // User's block list
      match /blocked/{blockedId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }
    }
    
    // 2. Global Scraped Outbreak Reports (Read-only public access)
    match /reports/{reportId} {
      allow read: if request.auth != null;
      allow write: if false; // Only backend service account can write reports
    }
    
    // 3. AI Chats Collection Rules
    match /chats/{chatId} {
      allow read, write: if request.auth != null && resource.data.user_id == request.auth.uid;
      allow create: if request.auth != null && request.resource.data.user_id == request.auth.uid;
      
      match /messages/{messageId} {
        allow read, write: if request.auth != null && get(/databases/$(database)/documents/chats/$(chatId)).data.user_id == request.auth.uid;
      }
    }
    
    // 4. Regional Groups Collection Rules
    match /groups/{groupId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null && request.resource.data.created_by == request.auth.uid;
      allow update: if request.auth != null && (request.auth.uid in resource.data.members || request.auth.uid in request.resource.data.members);
      
      match /messages/{messageId} {
        allow read: if request.auth != null;
        allow create: if request.auth != null && request.resource.data.sender_id == request.auth.uid;
      }
    }

    // 5. Peer-to-Peer Direct Chats Collection Rules
    match /direct_chats/{chatId} {
      // Allow read/write only if the authenticated user is a participant
      allow read, write: if request.auth != null && (request.auth.uid in resource.data.participants);
      allow create: if request.auth != null && (request.auth.uid in request.resource.data.participants);

      match /messages/{messageId} {
        allow read: if request.auth != null && (request.auth.uid in get(/databases/$(database)/documents/direct_chats/$(chatId)).data.participants);
        // Allows message creation and soft-deletes/edits
        allow create, update: if request.auth != null && request.resource.data.senderId == request.auth.uid;
        allow delete: if request.auth != null && resource.data.senderId == request.auth.uid;
      }
    }
  }
}
```
